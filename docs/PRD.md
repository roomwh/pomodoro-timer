# 식물이 자라는 뽀모도로 타이머 MVP PRD

## 핵심 정보

**목적**: 집중 시간 동안 화면 속 식물이 실시간으로 자라나는 시각적 리워드를 통해, 단조로운 타이머를 몰입 경험으로 전환한다.
**사용자**: 식물 키우기를 좋아하며 집중력 향상에 관심 있는 20~40대 1인 개발자, 재택근무자, 학생.

---

## 사용자 여정

```
1. 랜딩 페이지 (비로그인 상태)
   ↓ "시작하기" 버튼 클릭

2. 회원가입 페이지 또는 로그인 페이지
   - 회원가입: 이메일 확인(Confirm email)이 켜진 경우 "메일함을 확인하세요" 안내
     → 확인 메일 링크 클릭 → 세션 발급 → 로그인 처리
   - 로그인: 인증 성공 → 세션 발급
   ↓ 세션 발급 완료 → 자동 리디렉션
   ※ MVP에서는 Supabase 이메일 확인을 비활성화해 가입 즉시 로그인 처리하는 것을 기본 전략으로 한다.

3. 타이머 페이지 (메인 화면)
   ↓ 집중 시간 입력 (분 단위) → "타이머 시작" 클릭

4. 타이머 진행 중 (타이머 페이지 내 진행 상태)
   - 씨앗 → 새싹 → 꽃봉오리 → 만개 순으로 실시간 성장
   ↓
   [포기] → 식물 시들기 애니메이션 → 세션 "포기"로 저장 → 타이머 페이지 초기 상태로 복귀
   [완료] → 식물 만개 애니메이션 → 세션 "완료"로 저장 → 정원에 식물 추가

5. 정원 페이지
   - 완성된 식물 누적 목록 확인
   - 날짜별 집중 기록 열람
   ↓ "타이머로 돌아가기" 클릭 → 타이머 페이지로 이동
```

## 데이터 모델

### focus_sessions (집중 세션 기록)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| user_id | 세션 소유자 | → auth.users.id |
| duration_minutes | 설정한 집중 시간 (분) | integer (CHECK: 1~180 범위 제약 권장) |
| started_at | 타이머 시작 시각 | timestamp with time zone |
| completed_at | 타이머 종료 시각 (완료 또는 포기 시점) | timestamp with time zone |
| status | 세션 결과 상태 | enum: 'completed' / 'abandoned' |
| plant_type | 배정된 식물 종류 (랜덤 배정) | varchar |
| created_at | 레코드 생성 시각 | timestamp with time zone |

### garden_plants (정원 식물 목록)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| user_id | 정원 소유자 | → auth.users.id |
| session_id | 이 식물을 생성한 세션 | → focus_sessions.id |
| plant_type | 식물 종류 | varchar |
| grown_at | 정원에 심어진 날짜 | date |
| created_at | 레코드 생성 시각 | timestamp with time zone |

> **Row Level Security (RLS)**: 두 테이블 모두 본인 데이터만 접근하도록 operation별 정책을 명시한다.
> - `SELECT` / `UPDATE` / `DELETE`: `USING (user_id = auth.uid())`
> - `INSERT`: `WITH CHECK (user_id = auth.uid())` — 타인 명의로의 삽입을 차단하기 위해 반드시 필요.
>
> **데이터 모델 단순화 검토 (선택)**: `garden_plants`는 `focus_sessions` 중 `status = 'completed'` 레코드와 사실상 중복된다. MVP에서는 `garden_plants` 테이블을 두지 않고 `focus_sessions` 단일 테이블 + (선택적) DB View로 정원 뷰(F005)를 도출해 RLS 정책·삽입 트랜잭션·동기화 부담을 줄일 수 있다. 별도 테이블 유지 여부는 구현 시점에 결정한다.
>
> **무결성 한계 (MVP 수용)**: `status = 'completed'`를 클라이언트가 직접 삽입하므로 실제 집중 없이 완료를 위조할 여지가 있다. MVP에서는 수용하되, 성공 지표(완료율 등)의 신뢰도 한계로 인지한다.

## 타이머 신뢰성 설계 (핵심 요구사항)

브라우저는 백그라운드 탭에서 `setInterval`/`setTimeout`을 throttle(최소 1초, 다수 탭/모바일에서는 수 분 단위 또는 정지)하며, 모바일에서 화면 잠금/앱 전환 시 JS 실행이 통째로 suspend된다. 따라서 **틱 개수를 누적해 경과 시간을 세는 방식은 오차(drift)가 반드시 발생**한다.

### 절대 시각 기반 설계 (필수)

| 항목 | 설계 |
|------|------|
| **경과 시간 계산** | 틱 카운팅 금지. 타이머 시작 시 `startTime`(절대 시각)을 저장하고, 매 틱마다 `elapsed = Date.now() - startTime`으로 **재계산**한다. `progress = elapsed / total`. 이 방식은 백그라운드 복귀 시 자동으로 정확히 보정된다. |
| **렌더 틱** | UI 갱신용 `setInterval`은 표시 갱신 용도일 뿐, 시간의 source of truth가 아니다. |
| **백그라운드 복귀 보정** | `visibilitychange` / `focus` 이벤트에서 `elapsed`를 재계산하고, `elapsed >= total`이면 즉시 '완료'로 확정한다. |
| **완료 확정** | 탭/화면이 잠긴 동안에는 정확한 "0 도달 순간" 콜백이 못 뜬다. 복귀 시점 재계산으로 완료를 판정한다. MVP는 알림/소리가 없어(범위 제외) 이 한계가 치명적이지 않다. |

### 알려진 한계 (MVP 수용)

모바일에서 화면을 잠그면 애니메이션은 멈추지만, 복귀 시 정확한 성장 단계로 점프(보정)한다. 백그라운드 알림이 MVP 범위 밖이므로, 사용자가 다른 앱으로 이동하면 완료 피드백을 즉시 받지 못한다. 이는 PWA/푸시가 있어야 해결되며 **명시적 한계로 수용**한다.

## 식물 성장 애니메이션 구현 방향

### 기본 전략

타이머 경과 시간 비율(`progress = elapsed / total`, 0.0 ~ 1.0)을 CSS 변수에 직접 바인딩해 실시간 성장을 표현한다. `elapsed`는 위 "타이머 신뢰성 설계"에 따라 타임스탬프 차분으로 계산한다.

### 성장 단계 분기 (progress 기준)

| 단계 | progress 범위 | 표현 |
|------|--------------|------|
| 씨앗 | 0.00 ~ 0.10 | 작은 원형 씨앗 SVG |
| 새싹 | 0.10 ~ 0.40 | 줄기 높이 점진 증가, 잎 2장 등장 |
| 꽃봉오리 | 0.40 ~ 0.80 | 꽃봉오리 크기 증가, 색상 채색 |
| 만개 | 0.80 ~ 1.00 | 꽃잎 펼침, 빛나는 효과 |
| 시들기 | 포기 시 | 채도 감소(grayscale) + 아래로 기울어지는 트랜지션 |

### 구현 방식

```
SVG 기반 (권장):
- 각 성장 단계를 SVG 패스/도형으로 정의
- progress 값을 CSS 변수(--progress)에 바인딩하고, SVG 속성(height, opacity, transform)은 이 변수에 연동
- CSS transition으로 부드러운 단계 전환

CSS 애니메이션 보조:
- 만개 시 꽃잎 펼침: CSS keyframe animation (@keyframes bloom)
- 시들기: CSS transition + filter: grayscale(1) + transform: rotate(-15deg)
```

> **성능 주의**: 매 프레임마다 React state를 갱신해 식물을 리렌더하지 않는다. 카운트다운 숫자(MM:SS)는 1초 간격으로만 갱신하고, 식물의 부드러운 성장은 `progress`를 **CSS 변수(`--progress`)에 써서 CSS transition에 위임**한다. 이렇게 하면 불필요한 리렌더 없이 GPU 가속(transform/opacity)으로 60fps 부드러움을 확보한다.

### 식물 종류 (MVP: 3종 랜덤 배정)

- 튤립 (빨간색)
- 해바라기 (노란색)
- 선인장 (초록색)
